Problem 14 LONG (2.5 pts)

  • One of the two most challenging ROSALIND problems.

  • Builds on number 12 GRPH. (Review “Assembly_Algorithms.pptx” on Week03 Sakai for inspiration.)

  • The greedy shortest common superstring algorithm will return the correct answer given constraints of this problem.

  • Thus, start by finding the two strings with the longest overlap, replace these two strings with their merged string, then search for the next longest overlap, and repeat until you only have one merged string.

  • Consider writing functions to find the overlap between two strings and the pair of strings with the maximum overlap in a list of strings.

Problem 15 GASM (2.5 pts)

  • The other most challenging ROSALIND problem.

  • Builds on number 13 DBRU. (Review “Assembly_Algorithms.pptx” on Week03 on Sakai for inspiration.)

  • You don’t know which value of k gives exactly two directed cycles, so iterate until you find the right value of k.

  • For each value of k starting with the read length, build the k-1 de Bruijn graph and try to build just 2 directed cycles that consume all of your edges. For example, here are the k=5 and k=4 graphs of the small Sample Dataset. For this sample case, k=5 has no cycles and k=4 gives exactly 2 directed cycles.

  • Also, the problem asks for one cyclic superstring, so you shouldn’t write both that you find.

Visualizing graphs with graphviz

  • You may find it helpful to visualize graphs as you develop your solutions to Problems 12-15.
  • Install the graphviz software first. Then install the graphviz module: pip install graphviz or pip3 install graphviz should work. (Or use the class servers, graphviz is installed on both.)
  • Here’s how to visualize the de Bruijn graph from the Sample Output from Problem 13:
import graphviz
dbg = graphviz.Digraph('dBgraph', format='png')
with open("sample_output_13.txt", 'r') as f:
    for line in f:
        e1, e2 = line.rstrip().split(', ')
        e1 = e1[1:]
        e2 = e2[:-1]
        dbg.edge(e1, e2)
dbg.render()
